## Suite — Pont Facteur → Formateur créé et pushé

### Système déployé : Support technique automatique via emails entrants

**Fichier créé :** `/data/.openclaw/plugin-skills/facteur/bridge-formateur.js`
- Module pont entre le Facteur (lecture emails Gmail) et le Formateur (réponse technique)
- **CLI utilisable** : `node bridge-formateur.js classify <texte>` / `answer <leadId> <question>` / `process <from> <subject> <body>`
- **3 exportations** : `classifierEmailTechnique()`, `getFormateurAnswer()`, `traiterEmailTechnique()`

### Algorithme de classification
- 18 indices techniques (bug, erreur, ne marche pas, comment faire, mot de passe, SEO, paiement...)
- 10 indices non techniques (bonjour, merci, tarif, devis, nouveau projet...)
- Score technique > 5 ET score technique > score non technique → question technique
- Edge cases gérés : "ne s'affiche pas" (5pts), "comment modifier" (mixé si contexte technique)
- Fallback : phrases courtes + ton commercial = non technique

### Routing automatique
1. `classifierEmailTechnique(subject, body)` → `{ isTech, score, question, reason }`
2. `trouverProjetParEmail(fromEmail)` → cherche dans Supabase (table `agent_memory`, categorie `projet_livre`) le projet correspondant
3. `getFormateurAnswer(projet, question)` → appelle Claude Sonnet avec contexte client (nom, CMS, fonctionnalités)
4. `stockerFAQ(projet, question, reponse)` → enregistre Q/R dans `memory_semantic` (categorie `lecon_formateur`, action `faq-repondu`) pour enrichir les futures FAQ
