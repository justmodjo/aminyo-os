
## 2026-05-31 - AminyoOS créé, push bloqué par permissions GitHub

**Création du dashboard AminyoOS :**
- Projet Next.js 16 + React 19 + TypeScript + TailwindCSS v4 + Supabase v2 + Lucide React
- 6 pages : Dashboard, CRM, Projects, Tasks, Maintenance, Payments
- Build réussi ✅
- Design system : fond `#f8f7fc`, accent violet `#7c3aed`, `.card` et `.card-interactive`
- Connecté aux tables Supabase existantes

**Problème push GitHub :**
- Repo `justmodjo/aminyo-os` créé sur GitHub (par Amine)
- Token stocké (`MINO_GITHUB_TOKEN`) est un fine-grained PAT (`github_pat_...`) sans droits d'écriture
- Échec push via HTTP (403) et via API REST Contents (permissions insuffisantes)
- Bloqué : besoin d'un PAT classique (`ghp_...`) avec scope `repo`, ou permission `Contents: write` ajoutée au fine-grained PAT

**Leçon :** Toujours vérifier les permissions du token (write vs read) avant de commencer un push. Les fine-grained PATs nécessitent une permission explicite par repo.
